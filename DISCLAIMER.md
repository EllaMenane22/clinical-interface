# Disclaimer

## Synthetic data

The CSV files in the `data/` folder are **synthetic dummy data**. They do not
contain any real patient information. The three patients, their Cobb angles,
convexity directions, end vertebrae and pad measurements are all made up and do
not correspond to any real person.

Because this repository is public, no real clinical data has been uploaded, in
order to protect the privacy of the patients in the study. The actual project
was validated against a separate, fully anonymised retrospective clinical
dataset, which is not shared here.

## This is not a clinical tool

This interface is a research proof of concept. It is not a validated medical
device and must not be used to make clinical decisions.

## Required FE model files (not included)

The app does not generate spine models itself; it reads model files produced by
the separate finite element pipeline. To run a prediction it expects, for each
patient, a folder at:

```
Automated models/Updated automated/{PID}/
```

containing that patient's FE model files:

- `{PID}.dat` (the Marc input deck)
- `{PID}.t16` (the solved baseline result)
- `{PID}_model_metadata.csv`
- `{PID}_spine.csv`

These are generated by the FE workflow and are **not** included in this
repository. Without them the interface will open and display the patient list,
but it cannot run a model or return a correction.

## Marc / py_post

Running a prediction also requires a local MSC Marc Mentat 2024.1 installation,
including its `py_post` library. `py_post.pyd` is proprietary MSC software and is
not included here; it must come from your own licensed Marc install.
